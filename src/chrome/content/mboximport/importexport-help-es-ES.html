Formato de fecha personalizado: basado en la implementación de TDohan strftime ()

Un puerto JavaScript de strftime () , una función para formatear la fecha y la hora.

Especificaciones de conversión:

Secuencia Descripción
%a Nombre abreviado del día de la semana.
%A Nombre completo del día de la semana.
%b Nombre abreviado del mes.
%B Nombre completo del mes.
%c Representación preferida de fecha y hora (UTC) para el entorno local actual.
%C Número de siglo (año / 100) como un entero de 2 dígitos.
%d Día del mes como número decimal (rango 01 a 31).
%e Día del mes como número decimal (rango 1 a 31).
%F Formato de fecha ISO 8601 (equivalente a % Y-% m-% d ).
%G ISO 8601 año basado en la semana con siglo como número decimal. El año de 4 dígitos corresponde al ISO número de semana (consulte % V ). Tiene el mismo formato y valor que % Y , excepto que si el número de semana ISO pertenece al año anterior o siguiente, ese año se usa en su lugar.
%g Como % G , pero sin siglo, es decir, con un año de 2 dígitos (00-99).
%H Hora como número decimal usando un reloj de 24 horas (rango 00 a 23). Consulte también % k .
%I Hora como número decimal usando un reloj de 12 horas (rango 01 a 12). Consulte también % l .
%j Día del año como un número decimal (rango 001 a 366).
%k Hora como un número decimal usando un reloj de 24 horas (rango de 0 a 23). Consulte también % H .
%l Hora como un número decimal usando un reloj de 12 horas (rango 1 a 12). Consulte también % I .
%m Mes como un número decimal (rango 01 a 12).
%n Mes como un número decimal (rango 1 a 12).
%M Minuto como un número decimal (rango 00 a 59).
%p "AM" o "PM" según el valor de tiempo dado. El mediodía se trata como "PM" y la medianoche como "AM".
%P Como % p pero en minúsculas ("am" o "pm").
%s Número de segundos desde la Época, 1970-01-01 00:00:00 +0000 (UTC).
%S Segundo como un número decimal (rango 00 a 59).
%u Día de la semana como decimal (rango 1 a 7), siendo el lunes 1. Ver también % w .
%V Número de semana ISO 8601 del año actual como un número decimal (rango 01 a 53), donde la semana 1 es el primera semana que tiene al menos 4 días en el año nuevo (es decir, el primer jueves).
%w Día de la semana como decimal (rango de 0 a 6), siendo el domingo 0. Véase también % u .
%x Representación de fecha preferida para la configuración regional actual sin la hora.
%X Representación de hora preferida para la configuración regional actual sin la fecha.
%y Año como un número decimal sin un siglo (rango 00 a 99).
%Y Año como un número decimal que incluye el siglo.
%z La zona horaria numérica + hhmm o -hhmm (es decir, el desplazamiento de hora y minuto de UTC).
%Z Nombre o abreviatura de zona horaria.

Notas de compatibilidad:

  • % c : la cadena formateada es ligeramente diferente
  • % D : no implementado (use % m /% d /% y o % d /% m /% y ) < / li>
  • % e : no se agrega espacio
  • % E - no implementado
  • % h - no implementado (use % b )
  • % k : no se agrega espacio
  • % n - como % m , pero sin cero a la izquierda (use \ n para nueva línea)
  • % O - no implementado
  • % r - no implementado (use % I:% M:% S% p )
  • % R - no implementado (use % H:% M )
  • % t - no implementado (use \ t )
  • % T - no implementado (use % H:% M:% S )
  • % U - no implementado
  • % W - no implementado
  • % + - no implementado
  • %% - no implementado (use % )
